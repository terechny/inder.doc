
## Классы и объект

##### Объявление класса и конструктор

  - Объявление класса: Синтаксис `class MyClass {}`. 
    Шаблон для создания объектов. 
    Содержит свойства (переменные) и методы (функции), описывающие состояние и поведение сущности.

  - Конструктор (`__construct`): Магический метод. 
    Автоматически запускается в момент создания объекта через `new`. 
    Используется для инициализации начальных свойств.

##### Область видимости и контексты

  - `public` — доступны везде. 
  - `protected` — доступны внутри класса и его наследников.
  - `private` — доступны строго внутри самого класса.

  - `$this` указывает на текущий конкретный объект. 
  - `self` ссылается на сам класс (для статики и констант).
  - `parent` обращается к статическим методам или конструктору родительского класса.


##### Статические элементы и константы

- Статические свойства и методы: Объявляются ключевым словом `static`. 
  Принадлежат самому классу, а не его объектам. 
  Вызываются через оператор `::` без создания экземпляра класса.

- Константы класса: Объявляются через `const`. 
  Всегда публичны по умолчанию (в PHP 7.1+ можно задавать видимость). 
  Не могут быть изменены после объявления. Доступны через `ClassName::CONST_NAME`.

##### Особенности объектов

* Передача объектов: В PHP объекты передаются по ссылке (технически — передается копия указателя на объект). 
  Изменение свойств объекта внутри функции изменит их и снаружи этой функции.

## Наследование

* Наследование и переопределение:

- Что такое наследование: Механизм расширения классов через class `Child extends Parent`. 
   Дочерний класс автоматически получает все `public` и `protected` свойства и методы родителя.

- Переопределение методов (`Override`): 
  Изменение логики родительского метода в дочернем классе. 
  Метод объявляется заново с тем же именем. 
  С PHP 8.3 для явного контроля добавлена кодовая отметка #[Override].

- `parent::__construct()`: Явный вызов конструктора родителя из конструктора наследника. 
   Необходим, если дочерний класс переопределил конструктор, 
   но должен сохранить базовую инициализацию.

* Ограничения и абстракция

- `final` (запрет наследования/переопределения): Модификатор защиты. 
- `final class` запрещает создавать от него наследников. 
- `final method` внутри обычного класса запрещает переопределять этот метод в дочерних классах.

- `abstract` (абстрактные классы): Шаблоны, от которых нельзя создать объект напрямую (`new`). 
   Могут содержать абстрактные методы без тела, 
   которые наследники обязаны реализовать самостоятельно.

- Множественное наследование (нет в PHP): 
  Класс в PHP может наследоваться строго от одного родителя. 
  Ограничение обходится реализацией множества интерфейсов 
  или горизонтальным переносом кода через трейты (`Traits`).

## Интерфейсы

* Суть интерфейсов

- Что такое интерфейс: Чистый контракт поведения, объявляемый через interface. 
  Содержит только публичные методы без тела (`function name()`). 
  Класс, использующий интерфейс, обязан реализовать все его методы через implements.

* Возможности синтаксиса

- Множественная реализация: 
  Один класс может реализовывать несколько интерфейсов одновременно 
  (`class MyClass implements InterfaceA, InterfaceB`). 
  Это гибко заменяет отсутствие множественного наследования классов.

- Наследование интерфейсов: 
  Интерфейсы могут наследоваться друг от друга с помощью `interface ChildInterface extends ParentInterface`. 
  Дочерний интерфейс объединяет требования родительского и свои собственные.

- Константы в интерфейсах: 
  Интерфейсы могут содержать константы. 
  Они всегда публичны (`public`), и реализующие классы не могут их переопределять. 
  Обращение идет через имя интерфейса: `InterfaceName::MY_CONST`.

* Отличия и стандарты

- Интерфейс vs Абстрактный класс: 
  Интерфейс задает только поведение (контракт) и не может иметь свойств (переменных) или логики в методах. 
  Абстрактный класс задает идентичность (сущность), может иметь свойства, конструктор и готовую логику.

* PSR интерфейсы: Стандарты рекомендаций PHP (PHP FIG). 
  Общие интерфейсы для экосистемы, обеспечивающие совместимость разных библиотек. 

- PSR-3 — Logger Interface (LoggerInterface)
- PSR-6 — Caching Interface (CacheItemPoolInterface)
- PSR-7 — HTTP Message Interface (RequestInterface, ResponseInterface)
- PSR-11 — Container Interface (ContainerInterface)
- PSR-15 — HTTP Server Request Handlers (RequestHandlerInterface, MiddlewareInterface)
- PSR-16 — Simple Cache (CacheInterface)
- PSR-18 — ClientInterface



## Абстрактные классы

* Что такое абстрактный класс: 
  Базовый класс-шаблон, создаваемый через `abstract class`. 
  От него нельзя создать объект напрямую с помощью `new`. 
  Он служит фундаментом для расширения другими (конкретными) классами.

* Абстрактные методы: 
  Сигнатуры методов без тела (`abstract public function run();`). 
  Они декларируют обязательное поведение. 
  Любой некорректный дочерний класс обязан полностью реализовать эти методы 
  с тем же или более мягким уровнем видимости.

* Видимость и отличия

- Абстрактный класс vs Интерфейс: 
  Абстрактный класс описывает объектную сущность (чем является объект), 
  может иметь свойства, состояние и готовую логику. 
  Интерфейс описывает строго роль/контракт (что умеет делать объект) и не имеет свойств.

- `protected` в абстрактных классах: Механизм инкапсуляции для иерархии. 
   Позволяет объявлять абстрактные методы или готовые свойства, 
   скрытые от внешнего кода, но доступные только внутри самого абстрактного класса и классов-наследников.

* Паттерны проектирования

- Template Method Pattern: 
  Шаблон проектирования, использующий абстрактный класс. 
  Задает жесткий каркас алгоритма в финальном или обычном методе родителя, 
  а выполнение отдельных шагов делегирует абстрактным методам, которые переопределяют наследники.


## Трейты (Traits)

* Суть и синтаксис трейтов

- Что такое трейт: 
  Механизм повторного использования кода (`trait MyTrait`). 
  Позволяет горизонтально внедрять готовые методы в любые классы через `use MyTrait`. 
  Помогает обходить ограничение одиночного наследования.

- Несколько трейтов: 
  Один класс может подключать множество трейтов одновременно, 
  перечисляя их через запятую: `use TraitA, TraitB;`. Все методы этих трейтов становятся частью класса.

* Разрешение конфликтов и видимость

  - Конфликт имён методов: 
    Если два трейта имеют методы с одинаковым именем, возникает фатальная ошибка. 
    Конфликт разрешается внутри класса вручную через операторы `insteadof` 
    (выбрать один метод) и `as` (создать псевдоним).

  - Изменение видимости методов: 
    Оператор `as` позволяет менять модификатор доступа метода трейта при его импорте в класс. 
    Пример: `use MyTrait { myMethod as private; }` делает публичный метод трейта приватным для класса.

* Продвинутые возможности

  - Трейты с абстрактными методами: 
    Трейт может объявлять абстрактные методы (`abstract public function log();`). Это обязывает любой класс, использующий данный трейт, реализовать эти методы самостоятельно.
  - Трейты со свойствами: Трейты могут содержать свойства. Если класс объявляет свойство с тем же именем, его тип и начальное значение должны строго совпадать со свойством из трейта, иначе PHP выдаст ошибку.
  - Трейты в трейтах: Трейты могут состоять из других трейтов. При объявлении одного трейта можно использовать ключевое слово use для включения в него методов из другого трейта.

## Магические методы

* Инициализация и уничтожение

- `__construct` автоматически запускается при создании объекта (`new`). 
- `__destruct` срабатывает, когда на объект больше нет ссылок или во время завершения работы скрипта, очищая ресурсы.

* Магические свойства (Перегрузка свойств)

- `__get` и `__set`: Срабатывают при попытке обратиться к несуществующим или скрытым (`private/protected`) свойствам. `__get` читает значение, `__set` записывает его в обход прямой видимости.

- `__isset` и `__unset`: Вызываются автоматически при использовании функций контроля. `__isset` срабатывает при вызове `isset()` или `empty()` на скрытом свойстве, а `__unset` срабатывает при вызове `unset()`.


* Магические методы (Перегрузка методов)

- `__call` и  `__callStatic`: Перехватывают вызовы недоступных методов. `__call` обрабатывает вызов обычного метода у объекта, а `__callStatic` обрабатывает вызов статического метода у класса.


* Преобразование и вызов

- `__toString`: Автоматически преобразует объект в строку. Срабатывает, когда объект выводят через `echo`, `print` или используют внутри строковых операций. Должен возвращать строго тип `string`.

- `__invoke`: Позволяет использовать сам объект как обычную функцию. Срабатывает при попытке вызвать экземпляр класса со скобками: `$object()`. Напрямую превращает объект в тип `callable`.

##### Клонирование и сериализация

- `__clone`: Вызывается автоматически при копировании объекта через ключевое слово `clone`. 
  Используется для создания «глубокой копии», например, чтобы склонировать вложенные объекты, а не просто скопировать ссылки на них.

- `__sleep` и `__wakeup`: Управляют сериализацией.

- `__sleep` вызывается перед `serialize()`, определяя массив свойств для сохранения. 
- `__wakeup` запускается при `unserialize()` для восстановления ресурсов (например, подключения к БД).

## Статическое связывание (Late Static Binding)

##### Связывание в наследовании

* `self` ссылается на класс, где метод определён. `static` — на класс, где метод вызван (позднее связывание).

* `self` vs `static`: 
  - `self` реализует раннее связывание и всегда указывает на тот класс, где написан код. 
  - `static` реализует позднее статическое связывание (`Late Static Binding`) 
    и указывает на класс, который был фактически вызван в рантайме.

* `static::` в методах: Используется для вызова статических методов или свойств, 
   позволяя дочерним классам переопределять их. PHP определит нужный метод в зависимости от того, у какого класса был вызван контекст.

##### Инстанцирование объектов

* `new static` vs `new self`: 

  - `new self` всегда создает экземпляр базового класса, где написана эта строка. 
  - `new static` динамически создает объект того дочернего класса, который вызвал этот метод (полезно для паттернов `Factory` и `Singleton`).

* `parent::` с `static`: Ключевое слово `parent::` перенаправляет вызов в родительский класс, 
   но сохраняет информацию о вызвавшем классе (`static`), 
   если внутри родительского метода используется позднее статическое связывание.

##### Устаревшие инструменты и проблемы

* `get_called_class()`: Функция возвращала имя класса, в котором был вызван статический метод. 
   С PHP 8.0 полностью устарела, так как вместо нее используется встроенная языковая конструкция `static::class`.

* Проблемы с `Late Static Binding`: 
  Главная проблема — непредсказуемость доступа к приватным свойствам. 
  Если в родителе вызвать `static::$prop`, а свойство объявлено как `private`, 
  PHP выбросит ошибку, так как дочерний класс не имеет к нему доступа.

* `get_called_class()` возвращает имя вызывающего класса (устарела в PHP 8.0). 
  Используй `static::class` вместо неё. `static::class` возвращает полное имя класса с `namespace`. 

## Область видимости (Visibility)

##### Модификаторы и наследование

* `public`, `protected`, `private`: Уровни доступа. 
  - `public` открыт для всего приложения. 
  - `protected` виден только внутри текущего класса и его наследников. 
  - `private` строго изолирован внутри класса, где он объявлен.

* Области видимости в наследовании: 
  Дочерний класс полностью наследует и может использовать `public` и `protected` свойства/методы родителя.    
  Доступ к `private` элементам родительского класса полностью закрыт для наследников.

* Изменение видимости при переопределении: 
  При переопределении метода в дочернем классе нельзя сужать его область видимости, но можно расширять. 
  Из `protected` можно сделать `public`, но из `public` сделать `protected` или `private` запрещено.

##### Неизменяемость данных

* `readonly` свойства (PHP 8.1+): 
   Свойства, которые можно записать строго один раз (обычно в конструкторе). 
   После инициализации попытка изменить значение вызовет ошибку. 
   Требуют обязательного указания типа [2].

* `readonly` класс (PHP 8.2+): 
   Автоматически делает абсолютно все свойства класса неизменяемыми (readonly) 
   и запрещает динамическое добавление свойств. 
   От такого класса можно наследоваться только другим readonly классам.

* Константы класса и видимость: 
  Константы (`const`) поддерживают модификаторы `public`, `protected` и `private` (с PHP 7.1). 
  Они неизменяемы по умолчанию, а модификаторы управляют тем, откуда к ним можно обратиться.

  - `public` — для констант, используемых извне
  - `protected` — для констант, используемых в иерархии
  - `private` — для констант, используемых только в классе

##### Принципы проектирования

* Инкапсуляция — принцип ООП: 
  Концепция сокрытия внутреннего устройства объекта и защиты его данных от прямого изменения извне. 
  Реализуется через приватные/защищенные свойства и контролируемый доступ к ним через публичные методы (геттеры и сеттеры).
